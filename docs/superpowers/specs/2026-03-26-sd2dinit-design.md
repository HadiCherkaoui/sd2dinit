# sd2dinit Design Spec

## Context

Artix Linux (and other non-systemd distros) uses dinit as its init system, but many packages ship only systemd unit files. Converting these by hand is tedious and error-prone. **sd2dinit** automates this: it parses systemd `.service` files and generates equivalent dinit service files, runnable as a standalone CLI or as a pacman hook for automatic conversion on package install/upgrade.

## Architecture

Two-phase pipeline in a single crate (lib.rs + main.rs):

```
.service file → Parser → SystemdUnit → Converter → DinitService → Generator → dinit file
```

### File structure

```
sd2dinit/
├── Cargo.toml
├── src/
│   ├── lib.rs          # Re-exports public API
│   ├── parser.rs       # SystemdUnit, INI parsing, drop-in merging
│   ├── model.rs        # DinitService, DinitType, RestartPolicy, ConversionResult
│   ├── converter.rs    # SystemdUnit → ConversionResult (all mapping logic)
│   ├── generator.rs    # DinitService → String
│   ├── config.rs       # Config struct, TOML loading
│   ├── hook.rs         # Pacman hook logic (stdin reading, batch convert)
│   └── main.rs         # CLI (clap), colored output, orchestration
├── tests/
│   ├── parser_tests.rs
│   ├── converter_tests.rs
│   ├── generator_tests.rs
│   └── integration_tests.rs
└── hooks/
    └── sd2dinit.hook   # Alpm hook file
```

### Dependencies

- `clap` (derive) — CLI argument parsing
- `serde` + `toml` — config file
- `colored` — terminal output
- `anyhow` — error handling in binary
- `thiserror` — error types in library

## Data Structures

### SystemdUnit (parser output)

```rust
struct SystemdUnit {
    sections: HashMap<String, Vec<(String, String)>>,
    source_path: PathBuf,
    drop_in_paths: Vec<PathBuf>,
    parse_warnings: Vec<String>,   // malformed lines collected here
}
```

- Sections: `[Unit]`, `[Service]`, `[Install]` → ordered key-value pairs
- Multi-value keys (multiple `ExecStartPre=`) preserved as separate entries
- Empty-value assignments (`ExecStartPre=`) clear previous entries (drop-in override semantics)
- `parse_warnings`: malformed lines are collected here (not hard errors). `ParseError` is reserved for fatal issues (file not found, completely unparseable).

### DinitService (intermediate representation)

```rust
struct DinitService {
    name: String,
    source_path: PathBuf,          // provenance tracking
    service_type: DinitType,       // Process, BgProcess, Scripted
    command: Option<String>,
    stop_command: Option<String>,
    user: Option<String>,          // split from systemd User=
    group: Option<String>,         // split from systemd Group=
    working_dir: Option<PathBuf>,
    env_files: Vec<PathBuf>,       // multiple env-file lines supported
    pid_file: Option<PathBuf>,
    restart: RestartPolicy,        // Never, Always, OnFailure
    smooth_recovery: bool,
    restart_delay: Option<f64>,
    depends_on: Vec<String>,
    depends_ms: Vec<String>,
    waits_for: Vec<String>,
    logfile: Option<PathBuf>,
}

enum DinitType { Process, BgProcess, Scripted }
enum RestartPolicy { Never, Always, OnFailure }
```

- `user` + `group` stored separately; generator joins as `run-as = user:group` at output time
- `env_files` is a Vec: can contain both generated `.env` path and passthrough `EnvironmentFile=` paths
- `source_path` enables `# Generated by sd2dinit from ...` comment in output

### ConversionResult

```rust
struct ConversionResult {
    main_service: DinitService,
    pre_service: Option<DinitService>,
    post_service: Option<DinitService>,
    pre_script: Option<String>,        // wrapper script content for -pre
    post_script: Option<String>,       // wrapper script content for -post
    stop_script: Option<String>,       // wrapper for ExecStop + ExecStopPost
    env_file_content: Option<String>,  // generated .env from Environment=
    warnings: Vec<Warning>,
    should_enable: bool,               // from [Install] WantedBy/RequiredBy
}
```

### Warning

```rust
struct Warning {
    directive: String,      // e.g. "ProtectSystem"
    message: String,        // e.g. "sandboxing not supported, skipped"
    severity: Severity,     // Info, Warn, Error
}
enum Severity { Info, Warn, Error }
```

## Conversion Mapping

### Type mapping

| systemd `Type=` | dinit `type` | Notes |
|---|---|---|
| `simple` (default) | `process` | |
| `forking` + `PIDFile=` | `bgprocess` | `pid-file` set from `PIDFile=` |
| `forking` without `PIDFile=` | `process` | **Fallback** — warn that bgprocess needs pid-file, converting as process |
| `oneshot` | `scripted` | |
| `dbus` | `process` | Warn: dbus activation not supported |
| `notify` | `process` | Warn: notify not supported |

### Exec mapping

| systemd | dinit | Handling |
|---|---|---|
| `ExecStart=` | `command` | Replace known specifiers (`%n` → service name, `%N` → unescaped name), warn and remove unknown ones (`%i`, `%I`, etc.) |
| `ExecStop=` alone | `stop-command` | Direct map |
| `ExecStop=` + `ExecStopPost=` | `stop-command` → wrapper script | Script runs stop command then post-stop commands |
| `ExecStopPost=` alone | *skipped with warning* | dinit handles stop signals natively |
| `ExecStartPre=` (single, no `-`) | `-pre` scripted service, main `depends-on` it | |
| `ExecStartPre=` (single, `-` prefix) | `-pre` scripted service, main `depends-on` it | Command gets `\|\| true` in script |
| `ExecStartPre=` (multiple/mixed) | `-pre` scripted service + wrapper script | `-` prefixed lines → `\|\| true`; always `depends-on` |
| `ExecStartPost=` | `-post` scripted service, `waits-for` main | Same wrapping logic as pre |

**`-` prefix handling**: In systemd, `ExecStartPre=-/usr/bin/cmd` means failure is tolerated. In the generated wrapper script, these lines get `|| true` appended. The -pre service always uses `depends-on` (hard dep) because the `|| true` ensures the script itself succeeds even if the `-` prefixed command fails.

**Wrapper script format**: All generated `.sh` files must have `#!/bin/sh` shebang and `set -e` so that non-`-` prefixed commands properly fail the script on error.

### Dependency mapping

| systemd | dinit | Notes |
|---|---|---|
| `Requires=` | `depends-on` | |
| `Wants=` | `depends-ms` | |
| `After=` | `waits-for` | |
| `Before=` | *skipped* | Note: "Before= targets skipped (no equivalent in dinit)" |
| `Conflicts=` | *skipped* | Warning: "Conflicts= has no direct dinit equivalent" |

### Restart mapping

| systemd `Restart=` | dinit `restart` | Notes |
|---|---|---|
| `no` | `false` | |
| `always` | `true` | |
| `on-success` | `true` | **Lossy** — warn: "on-success maps to always-restart; dinit has no clean-exit-only mode" |
| `on-failure` | `on-failure` | |
| `on-abnormal` | `on-failure` | Closest equivalent |
| `on-abort` | `on-failure` | Closest equivalent |
| `on-watchdog` | `on-failure` | Closest equivalent |
| `RestartSec=` | `restart-delay` | Convert to seconds (float) |

When `Restart=` is `always` or `on-failure`, also set `smooth-recovery = true`.

### Environment mapping

| systemd | dinit | Handling |
|---|---|---|
| `Environment=KEY=VAL` | `env-file` → generated `.env` | Write `KEY=VAL` lines to `<service>.env` in output dir |
| `EnvironmentFile=/path` | `env-file` | Direct passthrough, path as-is |

Both can coexist: `env_files: Vec<PathBuf>` in IR. Generated `.env` comes first, then passthrough files.

### Other field mapping

| systemd | dinit |
|---|---|
| `User=` | `run-as` (user part) |
| `Group=` | `run-as` (group part) |
| `WorkingDirectory=` | `working-dir` |
| `PIDFile=` | `pid-file` |
| `WantedBy=` / `RequiredBy=` | Sets `should_enable` flag for install subcommand |

### Out of scope (warning + skip)

- Socket activation (`ListenStream`, `ListenDatagram`, etc.)
- Template/instance units (`name@.service`, `%i` specifiers)
- All sandboxing (`ProtectSystem`, `PrivateTmp`, `NamespaceOptions`, etc.)
- Slice/CGroup directives (`Slice=`, `CPUQuota=`, `MemoryMax=`, etc.)
- `ConditionPathExists` and similar conditionals
- Non-.service unit types (`.timer`, `.socket`, `.path`, `.mount`) — warn and skip at CLI level

## Parser

Hand-rolled INI parser handling systemd quirks:

1. **Sections**: `[SectionName]` lines start new sections
2. **Key-value**: `Key=Value` or `Key = Value` — trim surrounding whitespace
3. **Multi-value**: Multiple entries for same key preserved as separate pairs
4. **Continuation**: Lines ending with `\` join with the next line
5. **Empty-value reset**: `Key=` (no value) clears all prior entries for that key
6. **Comments**: Lines starting with `#` or `;` are ignored
7. **Drop-in merging**: After main file, scan `<unit>.d/*.conf` in alphabetical order, merge sections (later entries override, respecting empty-value reset)

Returns `Result<SystemdUnit, ParseError>`. Malformed lines are collected as warnings, not hard errors.

## Generator

`DinitService → String`. Simple serialization:

```
# Generated by sd2dinit from /usr/lib/systemd/system/sshd.service
type = process
command = /usr/sbin/sshd -D
run-as = root
restart = on-failure
restart-delay = 5
depends-on = network
waits-for = sshd-pre
env-file = /etc/dinit.d/sshd.env
```

- Only emits keys that have values (no empty lines for None fields)
- `run-as` joins `user:group` if both set, just `user` if only user, just `:group` if only group
- Provenance comment at top with source path

## CLI

```
sd2dinit convert <unit-file> [--output-dir /etc/dinit.d/] [--dry-run] [--force]
sd2dinit install <unit-file> [--enable] [--start] [--output-dir] [--dry-run] [--force]
sd2dinit hook
```

### convert
Parse → Convert → Write files to output dir:
- `<service>` (main dinit service file)
- `<service>-pre` (if ExecStartPre exists)
- `<service>-post` (if ExecStartPost exists)
- `<service>.env` (if Environment= exists)
- `<service>-pre.sh` / `<service>-post.sh` / `<service>-stop.sh` (wrapper scripts if needed)

`--dry-run`: Print all generated file contents to stdout without writing.
`--force`: Overwrite existing dinit files. Without this, skip existing files with a warning.

### install
Runs convert, then optionally:
- `--enable`: `dinitctl enable <service>`
- `--start`: `dinitctl start <service>`

### hook
Pacman hook mode. Reads file paths from stdin (one per line, NeedsTargets), filters for `.service` files, batch converts all. Uses config for output dir and ignored units.

### Exit codes
- 0: Clean success, all units converted
- 1: Partial success, some units had warnings
- 2: Failure, could not complete conversion

### Output
Colored terminal output via `colored`:
- Green: successful conversions
- Yellow: warnings (lossy mappings, skipped directives)
- Red: errors (parse failures, write failures)

## Config

`~/.config/sd2dinit/config.toml`:

```toml
output_dir = "/etc/dinit.d/"
ignored_units = ["systemd-tmpfiles-setup.service"]

[dependency_map]
"network-online.target" = "network"
"multi-user.target" = "boot"
```

All fields optional. Defaults:
- `output_dir`: `/etc/dinit.d/`
- `ignored_units`: empty
- `dependency_map`: built-in defaults for common systemd targets (overridable via config):
  - `"network-online.target" = "network"`
  - `"multi-user.target" = "boot"`
  - `"sysinit.target" = "boot"`
  - `"default.target" = "boot"`
  - Other names pass through as-is after stripping `.service` suffix

## Pacman Hook

`hooks/sd2dinit.hook`:

```ini
[Trigger]
Type = Path
Operation = Install
Operation = Upgrade
Target = usr/lib/systemd/system/*.service

[Action]
Description = Converting systemd units to dinit...
When = PostTransaction
Exec = /usr/bin/sd2dinit hook
NeedsTargets
```

Installed to `/usr/share/libalpm/hooks/sd2dinit.hook`.

## Error Handling

- **Library**: `thiserror` for typed errors (`ParseError`, `ConvertError`, `GenerateError`, `ConfigError`)
- **Binary**: `anyhow` wrapping library errors with context
- **Malformed input**: Warn and skip bad directives, never panic
- **File I/O**: Wrap with path context ("failed to read /usr/lib/systemd/system/foo.service")

- Parse real-world unit content: sshd, nginx, docker
- Multi-value keys, continuation lines, comments
- Empty-value reset (drop-in semantics)
- Drop-in directory merging
- Malformed lines produce warnings, not errors

### Unit tests — converter (`tests/converter_tests.rs`)
- Type mapping: simple, forking+pid, forking-no-pid fallback, oneshot, dbus, notify
- Exec mapping: single/multiple pre/post with `-` prefix handling
- Dependency mapping: Requires, Wants, After, Before (skipped), Conflicts (skipped)
- Restart mapping including on-success lossy warning
- Environment= generates env content, EnvironmentFile= passthrough
- ExecStop + ExecStopPost wrapper script generation
- Out-of-scope directives generate warnings

### Unit tests — generator (`tests/generator_tests.rs`)
- DinitService → output string correctness
- run-as user:group joining
- Provenance comment
- Only set fields emitted

### Integration tests (`tests/integration_tests.rs`)
- Full pipeline: unit file string → parse → convert → generate → assert output
- Sample services: sshd, nginx, docker, typical AUR service
- Edge cases: drop-in overrides, mixed pre/post, env + env-file combo

## Verification

1. `cargo build` — compiles without errors
2. `cargo test` — all unit and integration tests pass
3. Manual test: create a sample sshd.service, run `sd2dinit convert sshd.service --dry-run`, verify output
4. Manual test: `sd2dinit convert sshd.service --output-dir /tmp/dinit-test/`, inspect generated files
5. Verify wrapper scripts are executable and syntactically valid (`bash -n <script>`)
